# IT-GURU Products Admin

Панель управления каталогом товаров в рамках тестового задания.
Проект включает авторизацию, таблицу товаров, поиск, сортировку и локальное добавление товара.

## Технологический стек

- React 19 + TypeScript
- Vite
- React Router
- Zustand (сессия)
- React Hook Form + Zod (валидация формы логина)
- Axios (запросы к API)
- CSS Modules
- react-hot-toast

## API

- Авторизация: DummyJSON Auth (`/auth/login`)
- Товары: DummyJSON Products (`/products`, `/products/search`)

## Реализованный функционал

### Авторизация

- Валидация обязательных полей формы входа
- Обработка ошибок API (toast + сообщение в форме)
- Логика "Запомнить данные":
  - `localStorage` при включенном чекбоксе
  - `sessionStorage` при выключенном чекбоксе
- Защита маршрутов:
  - без токена доступ к `/products` закрыт
  - при токене `/login` редиректит на `/products`

### Таблица товаров

- Загрузка данных из API
- Прогресс-бар во время подгрузки
- Поиск товаров (API + фильтрация в UI)
- Сортировка по столбцам
- Хранение состояния поиска и сортировки в URL-параметрах
- Подсветка рейтинга ниже 3 красным
- Визуально оформленная таблица под макет

### Добавление товара

- Кнопка `Добавить` открывает модальную форму
- Поля: Наименование, Цена, Вендор, Артикул
- Локальное добавление без записи в API
- Toast при успешном добавлении

## Установка и запуск

```bash
npm install
npm run dev -- --host 127.0.0.1 --port 4273
```

Открыть в браузере:

- `http://127.0.0.1:4273/login`

Тестовые данные для входа:

- Логин: `emilys`
- Пароль: `emilyspass`

## Сборка

```bash
npm run build
npm run preview -- --host 127.0.0.1 --port 4373
```

## Примечание для локальной разработки

В dev-режиме включен fallback авторизации для тестовой пары `emilys / emilyspass`,
чтобы локальная демонстрация не блокировалась при временной недоступности внешнего DummyJSON Auth.
